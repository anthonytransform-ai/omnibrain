# OmniBrain Dashboard

## Start Here

1. Read `Vault/Project_Context.md`.
2. Read the latest note in `Vault/Daily_Logs/`, if one exists.
3. Read `Vault/System/_System_MOC.md` and `Vault/Features/_Features_MOC.md`.
4. Read `Vault/Definition_of_Done.md` and `Vault/Anti_Patterns.md`.
5. Read only the relevant directive in `Vault/OS/`.
6. Save important working notes to `Vault/_inbox/` before changing topic.

## Core Folders

- `Vault/System/`: durable project truth
- `Vault/Features/`: feature-specific knowledge
- `Vault/Plans/`: archived history
- `Vault/Daily_Logs/`: dated working notes
- `Vault/_inbox/`: temporary but recoverable working notes
- `Vault/OS/`: operating rules for the AI assistant
- `Vault/Agents/`: specialist review templates

## Reminder For The AI

Keep explanations plain. The human may be a non-coder. Do not ask them to perform technical chores that you can safely do yourself.

Use staged work for risky changes. Preserve useful notes, then enrich them instead of rewriting project memory from scratch.

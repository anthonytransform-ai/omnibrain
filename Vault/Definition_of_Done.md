# Definition of Done (QA Checklist)

Before executing a new feature or concluding a sprint, you must cross-reference your generated tasks against this standard checklist. Note any irrelevant items as "skipped".

1. **Routing / State Updates**: Are all necessary navigation routes or application states updated to reach the new feature?
2. **User Documentation**: Does the README or a relevant document need to be updated to explain this feature?
3. **Error Handling**: Are network failures or bad inputs explicitly caught and presented to the user?
4. **Mobile Responsiveness**: Is the feature explicitly designed to work on small screens without breaking the layout?
